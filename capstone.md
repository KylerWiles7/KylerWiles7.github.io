## 📖 Narratives
### 📁 SceneManager.cpp
SceneManager.cpp is the source code for the scene in my OpenGL project. SceneManager.cpp utilizes the textures, scene lights, objects, shaders, transformations and locations. I originally worked on this code in October 2025 at SNHU.

I chose this artifact for my ePortfolio because it showcases my ability to work with older software, the skills I used to enhance C++ code, my ability to enhance previous work and my understanding of complex source code. OpenGL was created in 1992 and most of the software systems used today are even older. This artifact shows that I am capable of using older software and also capable of adding meaningful enhancements that improve the base code. The code in this artifact is written in C++ and I was able to integrate a fully visualized scene using only C++ code. This showcases my skill with C++ and my ability to improve the project by fully visualizing the scene. This artifact also shows that I am able to revisit older projects and add meaningful additions to the original code. I think this is important if there is a need for you to back-track to a previous project. SceneManager.cpp is a large and complex source file, which is expected because it manages the entire scene. The size and complexity demonstrate my understanding to navigate and work with complex source code.

I improved this artifact by fully visualizing the scene. To clarify, I added a backdrop(night sky), environmental structures(trees and moon), roads, new scene lights, updated textures and a truck. I learned more about OpenGL that I missed the first time around taking CS-330. I learned more about how the shaders and lights interact with the textures. I was able to make the lighting look crisp and clear by adjusting their values based on what I learned. I had challenges with the shaders initially and it was hard to test them due to the way the software works. There is not a preview window that shows me what is happening to the scene when I modify my values. I had to launch the project each time I changed something to see if it looked good. I incorporated feedback as I was modifying this artifact by using the feedback to make  changes needed within my project. I met certain course outcomes in this artifact for my Capstone class at SNHU.

### 📁 ViewManager.cpp
ViewManager.cpp is the source code that handles viewing/navigating the scene. I worked on this artifact in October 2025 at SNHU and when I finished the course ViewManager.cpp was a very simple file.

I included this artifact because it showcases my ability to implement Data Structures and Algorithms, my ability to add a database to an existing project (SQLite) and my skills in C++ by adding new features to an existing project. ViewManager.cpp was only supposed to be used for viewing and navigating a scene in this project. I utilized Data Structures and Algorithms to convert my project to an investigator-style game. This allows the player to interact with objects in the scene in order to beat the game. The player starts with a tutorial that shows them the controls and the story of the game. This enhancement shows that I am able to take an artifact only meant for viewing/navigating and convert it into a story game with interaction. I think that SQL is very important to know as a programmer. Managing databases is a useful skill and this artifact shows that I am capable of adding and utilizing a database. When the player plays the game the source code is storing the player's progress inside the database. 

I learned more about OpenGL, C++ and SQL by working on this project. I learned how to implement new features into OpenGL using C++. By effect I learned more about C++ and how it interacts with OpenGL. I used C++ and OpenGL together in order to create a gameplay loop alongside full controls for the player. This taught me more about data structures and algorithms as well. I learned how to introduce a database to an existing project through SQL. I also learned more about utilizing a database within a project because it is integrated in my gameplay loop. I faced many challenges with this artifact because it had ambitious goals. I struggled getting my gameplay loop to work correctly due to the fact that it was implementing many different features at once. I also struggled with utilizing my database and it took me a couple days to get it working like I wanted it to. I incorporated feedback by making changes on my project based on the feedback I received. This enhancement improved this artifact because there is more functionality within the source code that benefits the user. This artifact met all the course outcomes for my Capstone class at SNHU.
